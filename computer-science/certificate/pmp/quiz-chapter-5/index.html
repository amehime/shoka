# 每日一练

  1. 在以下哪种情况下，应该采用观察的方法来收集项目相关方的需求？ {.quiz}
    - 相关方的数量很少时
    - 需要用渐进明细的理念来开发需求时
    - 相关方不愿或不能说明他们的需求时
    - 需要组建焦点小组时

  2. 项目有 500 多个客户，为了调研客户满意度，应该用以下哪个工具？ {.quiz}
    - 问卷调查
    - 访谈
    - 名义小组技术
    - 德尔菲技术

  3. 项目范围管理旨在确保 {.quiz}
    - 做尽可能多的工作，使项目尽可能完美
    - 只做为成功完成项目所需要的工作
    - 为客户提供额外的产品功能
    - 确定不包括在项目范围内的工作

  4. 为了减少想法的数量，集中关注大家投票得分最高的想法。可以使用以下哪个技术？ {.quiz}
    - 头脑风暴
    - 名义小组技术
    - 亲和图
    - 思维导图

  5. 在项目的环境中，“范围” 可以指 {.quiz}
    - 产品范围和工作范围
    - 产品范围和项目范围
    - 项目范围和管理范围
    - 项目范围和需求范围

  6. 下列哪个工具适用于定义跨职能的需求 {.quiz}
    - 焦点小组
    - 引导
    - 名义小组
    - 问卷调查

  7. 下列哪一项技术有助于减少数据偏见，并避免任何个人对成果产生不当影响？ {.quiz}
    - 访谈技术
    - 头脑风暴技术
    - 德尔菲技术
    - 风险审查技术

  8. 以下关于产品范围和项目范围的说法，哪个是正确的 {.quiz}
    - 项目范围服务于产品范围
    - 项目范围的变化必然引起产品范围的变化
    - 产品范围的变化必然引起项目范围的变化
    - 产品范围服务于项目范围

  9. 原型法体现了下列哪种理念 {.quiz}
    - 学习型组织
    - 相关方参与
    - 渐进明细
    - 客户至上

  10. 应该使用哪个技术对收集到的原始需求进行归类，把相似性的各种需求归为一个更大的需求？ {.quiz}
    - 名义小组技术
    - 思维导图
    - 流程图
    - 亲和图

  11. 用来衡量项目范围完成情况的文件是 {.quiz}
    - 项目管理计划
    - 项目范围说明书
    - 项目工作分解结构
    - 产品需求文件

  12. 新项目经理接管了一个处于执行阶段的项目。项目经理了解到客户没有提供正确的产品需求。项目经理应采用下列哪一项计划中的描述来分析。记录和管理需求？ {.quiz}
    - 范围管理计划
    - 变更管理计划
    - 配置管理计划
    - 需求管理计划

  13. 在收集需求过程中，需要逐一。深入地了解每一个相关方的具体需求。最好使用下列哪个技术？ {.quiz}
    - 访谈
    - 问卷调查
    - 焦点小组
    - 引导

  14. 项目经理正在实施一个公司范围的 IT 系统，该系统预期将对组织和其许多相关方产生重大影响，应该使用下列哪一项来收集需求？ {.quiz}
    - 问卷调查
    - 访谈
    - 原型法
    - 观察和交谈

  15. 项目团队开发了新产品。项目经理正在努力识别相关方的明确需求，并担心可能影响执行阶段的问题。应采取哪一个策略来尽可能减轻未来返工的风险？ {.quiz}
    - 焦点小组
    - 标杆对照
    - 原型制作
    - 名义小组

  16. 项目经理正在管理一个在组织中处于高优先级的电信项目。定义范围之后，项目团队下一步将进行下列哪一个过程？ {.quiz}
    - 创建 WBS
    - 制定预算
    - 制定人力资源管理计划
    - 定义活动

  17. 确认范围应该在 {.quiz}
    - 项目结束时
    - 项目开始时
    - 项目每个阶段期间
    - 计划编制期间进行一次

  18. WBS 典型的用途是？ {.quiz}
    - 去识别合适的项目出资人
    - 组织和定义项目的所有范围
    - 定义提供商提供给买主的包括的层次
    - 在工作元素分配给个人的时间记录

  19. 范围蔓延是指？ {.quiz}
    - 项目范围的合理延伸
    - 产品范围的合理延伸
    - 经过批准的项目范围或产品范围变更
    - 未得到控制的项目范围或产品范围扩大

  20. 以下哪一项定义了可交付成果及验收标准 {.quiz}
    - 项目章程
    - 工作分解结构（WBS）
    - 商业文件
    - 范围说明书

  21. 编制工作分解结构，必须遵守下列哪项规则？ {.quiz}
    - 100% 规则
    - 对称规则
    - 编号规则
    - 一次分解到位规则

  22. 通过下列哪个过程监督项目和产品的范围状态，管理范围基准的变更？{.quiz}
    - 规划范围管理
    - 定义范围
    - 确认范围
    - 控制范围

  23. 创建 WBS 的工具是？ {.quiz}
    - 产品分析
    - 分解
    - 头脑风暴
    - 滚动式规划

  24. 项目经理被任命管理一个现有项目，需要了解项目可交付成果。项目经理应该参考下列哪一份文件？ {.quiz}
    - 项目章程
    - 项目需求规范
    - 项目范围说明书
    - 项目进度表

  25. 下面哪项是对范围蔓延的最佳描述？ {.quiz}
    - 对批准范围的轻微变更
    - 项目交付过程中的范围变更
    - 对项目范围的不可控变更
    - 对项目范围的可控变更

  26. “工作分解结构” 中的 “工作” 是指 {.quiz}
    - 为实现项目目标必须开展的活动
    - 为实现可交付成果必须开展的活动
    - 具体的工作任务
    - 作为活动结果的工作产品或可交付成果

  27. 在创建 WBS 的过程中，需要使用以下哪个项目文件？ {.quiz}
    - 范围管理计划
    - 需求管理计划
    - 需求文件
    - 需求跟踪矩阵

  28. 确认范围过程的主要目的是什么？ {.quiz}
    - 确保项目可交付成果按时完成
    - 通过获得客户对每个可交付成果的及时验收，保证项目不偏离轨道
    - 确保可交付成果符合技术规范
    - 给人们提供一个发表不同意见的机会

  29. 对于要在未来远期才完成的可交付成果，因信息不充分而无法立即分解到位团队应当怎么处理？ {.quiz}
    - 请求高级管理层下达具体的分解指示
    - 为不影响 WBS 编制，而强行立即分解到位
    - 把可交付成果外包出去
    - 采用滚动式规划的方法

  30. 在创建工作分解结构时，过细的分解会导致 {.quiz}
    - 管理效率的提升
    - 数据汇总更容易
    - 工作实施效率下降
    - 团队成员责任更明确

  31. 以下哪个是控制范围过程的输出 {.quiz}
    - 工作绩效数据
    - 工作绩效信息
    - 工作绩效报告
    - 质量控制测量结果

  32. 发起人或客户主要通过何种方式完成对可交付成果的可接受性验收？ {.quiz}
    - 检查和决策
    - 检查和偏差分析
    - 决策和投票
    - 核对单

  33. 需求跟踪矩阵通常用于 {.quiz}
    - 规划范围管理
    - 定义范围
    - 创建 WBS
    - 控制范围

  34. 项目经理与项目相关方开会，收集需求并制作项目需求文档。项目经理下一步该怎么做？ {.quiz}
    - 制定需求基准，并获得关键关系人的批准
    - 与团队一起创建工作分解结构
    - 准备一份详细的项目范围说明书
    - 制定项目管理计划

  35. 下述关于面向可交付成果的分解层次，正确的说法是？ {.quiz}
    - 所有可交付成果有相同的分解层次
    - 所有可交付成果应该分解到最大层次
    - 不同的可交付成果可以有不同的分解层次
    - 所有可交付成果有不同的分解层次

  36. 以下哪一个可交付物最佳的定义和组织了项目范围？ {.quiz}
    - 项目章程
    - 项目范围说明书
    - WBS
    - 需求文件

  37. 下哪个文件定义了可交付成果和验收标准？ {.quiz}
    - 商业文件
    - 项目管理计划
    - WBS
    - WBS 词典

  38. 项目团队收集所有相关方需求并定义项目范围后，项目团队下一步应做什么？ {.quiz}
    - 定义分配给项目资源的活动
    - 确认项目范围，满足项目目标
    - 制定与项目可交付成果一致的进度计划
    - 将项目的可交付成果分解成可管理的工作内容

  39. 在执行一个系统集成项目期间，项目经理请求负责验收测试的资源，以便获得客户批准。客户对初始范围请求一项变更。项目经理首先应该怎么做 {.quiz}
    - 获得变更控制委员会（CCB）的批准
    - 更新变更管理计划
    - 评估对预算和时间的影响
    - 请求发起人批准

  40. 团队将项目工作和可交付成果分成较小且更易于管理的部分。团队应使用下列哪一项输入？ {.quiz}
    - 需求文件
    - 分解
    - 范围基准
    - 工作分解结构

  41. 下面哪个关于范围变更控制的说法是正确的？ {.quiz}
    - 范围变更控制从来都不需要和其它控制过程综合在一起
    - 控制成本是控制范围的一种有效的方式
    - 范围变更控制必须和其它控制过程综合在一起
    - 控制进度是控制范围的一种有效的方式

  42. 在什么阶段会参考需求跟踪矩阵？ {.quiz}
    - 启动
    - 执行
    - 收尾
    - 整个项目生命周期

  43. 项目团队召开一次会议，定义项目需求并收集各种意见。团队应使用什么技术来为这些意见排序，以便进行审查和分析？ {.quiz}
    - 头脑风暴
    - 名义小组技术
    - 亲和图
    - 思维导图

  44. 项目实施阶段，一位团队成员找到了项目经理，因为他不知道哪些工作需要他来完成。以下哪个文件对工作包做了详细描述？ {.quiz}
    - 工作分解结构词典
    - 活动清单
    - 项目范围说明书
    - 项目范围管理计划

  45. 项目经理正在努力识别一个大型项目的完整范围，项目经理应该审查什么？ {.quiz}
    - 项目进度计划
    - 工作分解结构（WBS）和排列活动顺序
    - 网络图
    - 项目章程和需求文件

  46. 在项目执行期间，一名新相关方询问项目的单一需求如何与公司的业务需求保持一致。项目经理应与相关方一起查阅下列哪一份文件 {.quiz}
    - 项目管理计划
    - 需求跟踪矩阵
    - 项目章程
    - 需求文件

  47. 客户要求你实施一个正式合同不包括的任务，你应该？ {.quiz}
    - 了解这个请求，并建议客户提交一个正式的变更请求
    - 作为将来业务机会的迹象，接受客户请求
    - 否决请求并向项目发起人汇报
    - 召集项目团队会议并重写项目范围说明书

  48. 为了更好的收集到项目中的需求，你可能需要和各个不同的相关方接触，以便更全面地获得需求的信息，收集需求需要参考以下文件，哪个是错误的： {.quiz}
    - 项目章程
    - 商业文件
    - 需求文件
    - 协议

  49. 一位受过训练的主持人正在引导大家进行互动式讨论，这是一个： {.quiz}
    - 访谈
    - 焦点小组
    - 问卷调查
    - 标杆对照

  50. 哪个过程对需求文件进行过滤，以便筛选出最终的项目需求： {.quiz}
    - 定义范围
    - 确认范围
    - 核实范围
    - 批准范围

  51. 项目经理和团队成员已经完成了项目范围说明书的内容，一位相关方认为，项目范围说明书包括的内容不应该有： {.quiz}
    - 可交付成果
    - 范围基准
    - 验收标准
    - 项目除外责任

  52. 关于工作包和规划包的说法，正确的是： {.quiz}
    - 工作包属于 WBS 中的组件，而规划包不属于
    - 控制账户可以有多个工作包或规划包
    - WBS 词典不对工作包或规划包做具体解释
    - 工作包进一步明确后就成了规划包

  53. 以下是访谈中记录的客户要求，哪一项可以作为项目的需求？ {.quiz}
    - 办公场所装修要高档
    - 软件操作要便捷
    - 成本控制在合同额的 ±10% 范围内
    - 买足够客户使用的车辆

  54. 以下哪一项明确地说明了哪些内容不属于项目范围的是？ {.quiz}
    - 项目的除外责任
    - 项目假设
    - 产品验收标准
    - 项目制约

  55. 在工作分解结构自上而下的细分中，将导致 {.quiz}
    - 估算准确度降低
    - 估算准确度增加
    - 管理工作本身的成本更低
    - 某些工作的遗漏
